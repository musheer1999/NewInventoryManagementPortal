--------------------------------------------------
VERY IMPORTANT: UNIQUE ID RULE
--------------------------------------------------
EVERY product MUST have one permanent uniqueID.

uniqueID is generated using:
- product name
- company
- productID (if available)

Examples:
- brake-pad-honda-1234
- engine-oil-castrol

uniqueID rules:
• Stored in Product table
• Unique across system
• Used for auto‑fill in BUY and SELL
• Never changes once created

--------------------------------------------------
DATABASE SCHEMAS
--------------------------------------------------

1) Product
- name (string, required)
- company (string, required)
- productID (string, optional)
- uniqueID (string, required, unique)
- quantity (number, required)
- averageBuyPrice (number, required)
- createdAt (date)

2) BuyBill
- dealerName (string, optional)
- billDate (date, required)
- items: [
    {
      productRef (ObjectId),
      uniqueID (string),
      buyPrice (number),
      quantity (number),
      totalCost (number)
    }
  ]
- totalBillAmount (number)

3) SellBill
- customerName (string, optional)
- customerPhone (string, optional)
- billDate (date, required)
- items: [
    {
      productRef (ObjectId),
      uniqueID (string),
      sellingPrice (number),
      quantity (number),
      profitPerUnit (number),
      totalProfit (number)
    }
  ]
- totalBillProfit (number)

--------------------------------------------------
PAGE 1: PURCHASE ENTRY (BUY INVENTORY)
--------------------------------------------------

Purpose:
Add inventory using SINGLE or MULTIPLE products in ONE bill.

Bill‑level fields:
- Bill Date (date picker, default = today)
- Dealer Name (optional)

Product rows (dynamic, multiple allowed):
- uniqueID (optional)
- name
- company
- productID (optional)
- MRP (optional)
- buy price
- quantity

Buttons:
- Add Another Product
- Remove Product
- Submit Bill

Auto‑Fill Logic:
• If uniqueID is entered:
  - Auto‑fill name, company, productID
• If first time product:
  - uniqueID can be skipped
  - System generates uniqueID automatically

Business Logic:
• Existing product:
  - Increase quantity
  - Recalculate average buy price:
    ((oldAvg × oldQty) + (newPrice × newQty)) / totalQty

• New product:
  - Create product
  - averageBuyPrice = buy price

• totalBillAmount = sum of all item totalCost
• Display currency as ₹ everywhere

--------------------------------------------------
PAGE 2: SELL ENTRY (SELL PRODUCTS)
--------------------------------------------------

Purpose:
Sell MULTIPLE products to ONE customer in ONE bill.

Bill‑level fields:
- Bill Date (date picker, default = today)
- Customer Name (optional)
- Customer Phone (optional)

Product rows (dynamic):
- uniqueID OR productID
- selling price
- quantity

Auto‑Fill (MANDATORY):
• On entering uniqueID or productID:
  - Auto‑fill name
  - Auto‑fill company
  - Auto‑fill average buy price
  - Validate available stock

Business Logic:
• Reduce inventory per product
• profitPerUnit = sellingPrice − averageBuyPrice
• totalBillProfit = sum of all item profits

--------------------------------------------------
PAGE 3: DASHBOARD
--------------------------------------------------

TAB 1: PROFIT
- Monthly profit graph (based on billDate)
- Current month profit (₹)
- Total yearly profit (₹)

TAB 2: INVENTORY
- Total inventory value:
  sum(averageBuyPrice × quantity)
- Show ONLY products where quantity > 0

Inventory table columns:
name | company | productID | uniqueID | quantity | avg buy (₹) | total value (₹)

Each row MUST be clickable.

BUY HISTORY TABLE (VERY IMPORTANT):

Edit/Delete Logic:

--------------------------------------------------
EDIT & DELETE RULES (CRITICAL)
--------------------------------------------------

EDIT BUY BILL:
• Update inventory for ALL items
• Recalculate average buy price

DELETE BUY BILL:
• Reverse inventory
• Prevent delete if stock goes negative

EDIT SELL BILL:
• Recalculate profit
• Update inventory

DELETE SELL BILL:
• Restore inventory
• Update profit totals
